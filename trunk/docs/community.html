User & Developer Community

Quartz is an open-source project, and is always on the lookout for feature ideas, enhancements, bug reports and all-around help. Find out how you can contribute (or get help) below.

User Forums

If you can't figure something out, just ask in our helpful forum. Quartz contributors and seasoned users are known to hang out here frequently!

Before posting a question, please double-check documentation (including JavaDOCs, the tutorial, and the docs in the distribution's "docs" directory). When you do post in the forum, please note the version of Quartz that you are using, information about the environment in which you are using Quartz (app server version, database version, etc.), and any logged information about the problem (full stack traces, etc.).

You may also want to search the thousands of historical forum messages from teh previous forum used on SourceForge.

Bug Reporting / Feature Requests / Patch Submissions

Quartz uses the most excellent JIRA software to track issues.
(We might be biased though... JIRA uses Quartz as its scheduling component).

Use JIRA to report or submit the following:

  • Bugs - please be fairly certain the issue is, in fact, a bug.
  • Feature/Improvement requests
  • Code Contributions (patches or new features)

Paid (Commercial) Support

Getting Involved

Donations of code/development time/user support/etc. are all welcome. Quartz development is hosted on Java.Net. Anyone is welcome to contribute in any way they wish - code, services, advice/suggestions, answering forum questions, etc.

Mail Lists: Join the development mail list "dev@quartz.dev.java.net" to participate in the development of Quartz.

CVS: Anyone may access to the latest source code via CVS.

Current Quartz source code con be found in the CVS repository on Java.Net, which you can browse via the web, or checkout with a CVS client.

Historical Quartz (1.4.5 and earlier) source code can be found in the CVS repository on SourceForge. You may browse the CVS tree or access it on Source Forge with via the project's cvs instruction page.
